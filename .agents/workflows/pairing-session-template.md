```markdown
## Agents To Use

## Requested Output

- Product brief
- UX spec
- Architecture plan
- Code change
- Test plan
- Review findings
- DX recommendation

## Done Criteria

- Customer-visible behavior:
- Accessibility expectations:
- Test expectations:
- Verification commands:
- Documentation or release notes:
```

## Recommended Starts

- New feature: start with Product CX Strategist.
- Existing feature with unclear UX: start with UX Interaction Designer.
- Implementation-ready work: start with React Architecture Agent.
- Code review or release readiness: start with React Quality Engineer.
- Inconsistent UI patterns: start with Design Systems Agent.
- Local development friction: start with DX Tooling Agent.
